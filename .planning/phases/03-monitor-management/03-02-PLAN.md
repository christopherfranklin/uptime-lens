    Create `components/monitors/monitor-panel.tsx` as a "use client" component using @base-ui/react Dialog for the slide-out panel.

    **MonitorPanel props:**
    ```typescript
    interface MonitorPanelProps {
      open: boolean;
      onOpenChange: (open: boolean) => void;
      monitor?: MonitorRow | null; // null/undefined = create mode, populated = edit mode
    }
    type MonitorRow = typeof monitors.$inferSelect;
    ```

    **Panel structure:**
    - `Dialog.Root` with `open` and `onOpenChange` props (controlled)
    - `Dialog.Portal` > `Dialog.Backdrop` (semi-transparent overlay, `fixed inset-0 bg-black/40 z-40`) + `Dialog.Popup` (slide-out from right: `fixed inset-y-0 right-0 w-full max-w-[480px] bg-background border-l border-border shadow-xl z-50 overflow-y-auto`)
    - CSS transition on the popup: translate-x-full when closed, translate-x-0 when open. Use `data-[state=open]` and `data-[state=closed]` attributes from Base UI or manage via the `open` prop with Tailwind transition classes.
    - `Dialog.Title`: "Create Monitor" or "Edit Monitor" based on mode
    - `Dialog.Close` button (X icon) in top-right corner

    **Form fields:**
    - **Name**: text input, required, max 255 chars
    - **Type selector**: three buttons/radio group for HTTP / TCP / SSL. In edit mode, display as read-only text badge (type locked after creation per CONTEXT.md).
    - **URL/host field** (dynamic based on type):
      - HTTP: `<input type="url" placeholder="https://example.com" />` label "URL"
      - TCP: Two fields -- `<input type="text" placeholder="example.com" />` label "Host" + `<input type="number" min="1" max="65535" placeholder="443" />` label "Port". Combine to "host:port" for the `url` field in FormData.
      - SSL: `<input type="url" placeholder="https://example.com" />` label "URL"
      - In edit mode, display URL/host as read-only text (locked after creation per CONTEXT.md).
    - **Expected Status Code**: number input, only shown for HTTP type, default 200
    - **Advanced section** (collapsed by default under a "Advanced" toggle button):
      - Timeout (ms): number input, default 10000
      - Check interval (seconds): number input, default 180

    **Pause/Resume toggle** (edit mode only):
    - Below the form fields, above the delete button
    - Show current status text ("This monitor is active" / "This monitor is paused")
    - Button: "Pause Monitor" (if active) / "Resume Monitor" (if paused)
    - Calls `toggleMonitorStatus` server action via form action with hidden monitorId input
    - Uses `useTransition` for pending state

    **Delete button** (edit mode only):
    - At the very bottom of the panel, separated by a border-t
    - Red destructive styling: `text-destructive border-destructive hover:bg-destructive/10`
    - Clicking opens the `DeleteDialog` (does NOT directly delete)

    **Form submission:**
    - Use `useTransition` + `startTransition` wrapping the server action call
    - On create: build FormData with name, url (combine host:port for TCP), type, expectedStatusCode, timeoutMs, checkIntervalSeconds. Call `createMonitor(formData)`. On success, close panel.
    - On edit: build FormData with id, name, expectedStatusCode, timeoutMs, checkIntervalSeconds. Call `updateMonitor(formData)`. On success, close panel.
    - Show error message if action returns `{ error }`.
    - Show loading spinner/disabled state on submit button while pending.

    **Input styling:** Match the established pattern from settings-client.tsx: `w-full rounded-lg border border-input bg-background px-3 py-2.5 text-sm text-foreground placeholder:text-muted-foreground focus:border-brand-500 focus:outline-none focus:ring-2 focus:ring-brand-500/20`

    Create `components/monitors/delete-dialog.tsx` as a "use client" component:

    **DeleteDialog props:**
    ```typescript
    interface DeleteDialogProps {
      open: boolean;
      onOpenChange: (open: boolean) => void;
      monitorName: string;
      monitorId: number;
      onDeleted: () => void; // callback to close parent panel
    }
    ```

    - Uses @base-ui/react Dialog (centered modal, NOT slide-out)
    - `Dialog.Popup`: centered, `max-w-md`, white background, rounded-xl, shadow
    - Title: "Delete {monitorName}?"
    - Body: "This will permanently delete this monitor and all its associated data, including check history and incidents. This action cannot be undone."
    - Two buttons: "Cancel" (closes dialog) and "Delete" (red bg, calls `deleteMonitor` server action with FormData containing monitorId)
    - Uses `useTransition` for pending state on delete button
    - On success, calls `onDeleted()` to close both delete dialog and the parent panel
