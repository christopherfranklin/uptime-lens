  <name>Task 1: Create monitor CRUD server actions with tests</name>
  <files>app/actions/monitors.ts, tests/monitors/actions.test.ts</files>
  <behavior>
    - createMonitor: valid HTTP input (name + URL + type "http") returns success, monitor inserted with userId from session
    - createMonitor: valid TCP input (name + host + port as URL field + type "tcp") returns success
    - createMonitor: valid SSL input (name + URL + type "ssl") returns success
    - createMonitor: missing name returns { error: "Name is required" }
    - createMonitor: missing URL/host returns { error: "URL is required" } (or host for TCP)
    - createMonitor: invalid type returns error
    - updateMonitor: updates name, expectedStatusCode, checkIntervalSeconds, timeoutMs for owned monitor
    - updateMonitor: rejects update when monitorId not owned by session user (returns { error: "Monitor not found" })
    - updateMonitor: does NOT update type or url fields (locked after creation)
    - deleteMonitor: deletes owned monitor, returns success
    - deleteMonitor: rejects deletion of non-owned monitor
    - toggleMonitorStatus: toggles "active" to "paused" and "paused" to "active"
    - toggleMonitorStatus: rejects toggle on non-owned monitor
    - All actions call verifySession() for auth
  </behavior>
  <action>
    Create `app/actions/monitors.ts` with "use server" directive. Import `db` from `@/lib/db`, `monitors` from `@/lib/db/schema`, `verifySession` from `@/lib/dal`, and `eq`, `and`, `desc` from `drizzle-orm`.

    Implement four exported async functions:

    1. `createMonitor(formData: FormData)` - Extract name, url, type from FormData. Validate: name required (1-255 chars), url required, type must be "http"|"tcp"|"ssl". For TCP, validate url contains host:port format with port 1-65535. For HTTP/SSL, validate URL starts with http:// or https://. Insert into monitors with userId from session. Call `revalidatePath("/dashboard")`. Return `{ success: true, monitorId: number }` or `{ error: string }`.

    2. `updateMonitor(formData: FormData)` - Extract id (number), name, expectedStatusCode, checkIntervalSeconds, timeoutMs, status from FormData. Verify ownership: select monitor where id AND userId match. If not found, return `{ error: "Monitor not found" }`. Update only allowed fields (NOT type, NOT url). Call `revalidatePath("/dashboard")`. Return `{ success: true }` or `{ error: string }`.

    3. `deleteMonitor(formData: FormData)` - Extract id. Verify ownership. Delete with `db.delete(monitors).where(and(eq(monitors.id, id), eq(monitors.userId, userId)))`. CASCADE handles related data. Call `revalidatePath("/dashboard")`. Return `{ success: true }` or `{ error: string }`.

    4. `toggleMonitorStatus(formData: FormData)` - Extract id. Verify ownership. Read current status. Set to opposite ("active" <-> "paused"). Call `revalidatePath("/dashboard")`. Return `{ success: true, newStatus: string }` or `{ error: string }`.

    Create `tests/monitors/actions.test.ts` following the existing test pattern (vi.mock for db and dal). Use the delegate pattern for vi.mock closures (established in Phase 2 decisions). Mock `verifySession` to return `{ user: { id: "test-user-id" } }`. Mock db operations (insert, update, delete, select). Mock `revalidatePath` from `next/cache`. Test all behaviors listed above.
  </action>

  </verify>
