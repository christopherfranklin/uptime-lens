1. Create `drizzle.config.ts` following the research pattern:
   ```typescript
   import "dotenv/config";
   import { defineConfig } from "drizzle-kit";

   export default defineConfig({
     out: "./drizzle",
     schema: "./lib/db/schema.ts",
     dialect: "postgresql",
     dbCredentials: {
       url: process.env.DATABASE_URL!,
     },
   });
   ```

2. Create `lib/db/schema.ts` with the COMPLETE schema from research. All 7 tables:

   **Enums:**
   - monitorTypeEnum: "http", "tcp", "ssl"
   - monitorStatusEnum: "active", "paused"
   - incidentStatusEnum: "ongoing", "resolved"
   - subscriptionStatusEnum: "trialing", "active", "past_due", "canceled", "unpaid"

   **Tables (follow research examples EXACTLY for column types and naming):**
   - `users`: id (identity PK), email (unique varchar 255), name (varchar 255 nullable), createdAt, updatedAt
   - `monitors`: id (identity PK), userId (FK -> users, cascade delete), name, url, type (enum), status (enum, default active), expectedStatusCode (default 200), checkIntervalSeconds (default 180), timeoutMs (default 10000), consecutiveFailures (default 0), lastCheckedAt (nullable), createdAt, updatedAt. Indexes on userId, status.
   - `heartbeats`: id (bigint identity PK), monitorId (FK -> monitors, cascade delete), status (integer), responseTimeMs (real), statusCode (integer nullable), error (text nullable), sslExpiresAt (timestamp nullable), checkedAt. Indexes on (monitorId, checkedAt) and checkedAt.
   - `heartbeatsHourly`: id (bigint identity PK), monitorId (FK -> monitors, cascade delete), hour (timestamp), totalChecks, successfulChecks, avgResponseTimeMs, minResponseTimeMs, maxResponseTimeMs. Unique index on (monitorId, hour).
   - `heartbeatsDaily`: id (bigint identity PK), monitorId (FK -> monitors, cascade delete), date (timestamp), totalChecks, successfulChecks, avgResponseTimeMs, minResponseTimeMs, maxResponseTimeMs, uptimePercentage. Unique index on (monitorId, date).
   - `incidents`: id (identity PK), monitorId (FK -> monitors, cascade delete), status (enum, default ongoing), startedAt, resolvedAt (nullable), cause (text nullable), acknowledgedAt (nullable). Indexes on monitorId, status, startedAt.
   - `subscriptions`: id (identity PK), userId (FK -> users, cascade delete, unique), stripeCustomerId, stripeSubscriptionId, stripePriceId, status (enum, default trialing), planMonitorLimit (default 10), currentPeriodStart, currentPeriodEnd, trialEndsAt, canceledAt, createdAt, updatedAt. Indexes on userId, unique indexes on stripeCustomerId, stripeSubscriptionId.

   IMPORTANT: Use explicit snake_case column names for ALL columns (e.g., `userId: integer("user_id")`). Use `integer().generatedAlwaysAsIdentity()` for PKs, NOT `serial()`. All timestamps use `{ withTimezone: true }`.

3. Create `lib/db/index.ts`:
   ```typescript
   import { drizzle } from "drizzle-orm/neon-http";
   import * as schema from "./schema";

   export const db = drizzle(process.env.DATABASE_URL!, { schema });
   export * from "./schema";
   ```

4. Create `tests/db/schema.test.ts` with the behavior tests listed above. Import from `lib/db/schema` and verify exports exist and have the correct structure. Use Drizzle's introspection APIs (e.g., checking table column definitions) where possible.
