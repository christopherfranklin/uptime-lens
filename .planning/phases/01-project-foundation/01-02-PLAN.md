
2. Create `app/api/email/test/route.ts`:
   - POST handler that sends a test email using the resend client
   - From: "Uptime Lens &lt;alerts@uptimelens.io&gt;" (locked by user decision)
   - Accept `{ to: string }` in the request body (email address to send to)
   - Subject: "Test email from Uptime Lens"
   - Body: Simple HTML confirming email infrastructure works, include a note about SPF/DKIM/DMARC
   - Return `{ success: true, id: data.id }` on success
   - Return `{ error: string }` with status 500 on failure
   - Important: If the domain is not yet verified in Resend, the email will fail. This is expected. The test route exists so the user can verify once DNS propagation completes. During development, the user can temporarily use "onboarding@resend.dev" as the from address for testing.

   - Test that the resend module exports a Resend instance
   - Test that the POST route handler exists and is a function
   - Do NOT make actual API calls in tests (no real email sends)
   - Mock the RESEND_API_KEY environment variable for the test

4. Run tests: `npx vitest run tests/email/resend.test.ts --reporter=verbose`
