<task type="auto">
  <name>Task 1: Install Better Auth and extend database schema with auth tables</name>
  <files>package.json, lib/db/schema.ts</files>
  <action>
1. Install Better Auth: `npm install better-auth`

2. Run `npx @better-auth/cli generate` to inspect what schema Better Auth expects. Do NOT blindly apply the output -- the existing `users` table uses integer auto-increment IDs, not string UUIDs.

3. Extend `lib/db/schema.ts` with Better Auth tables. The critical challenge is ID type reconciliation:
   - The existing `users` table uses `integer().primaryKey().generatedAlwaysAsIdentity()`.
   - Better Auth expects string IDs by default. However, Better Auth's Drizzle adapter works with whatever schema you provide as long as the table names and column names match.
   - Add an `emailVerified` boolean column to the existing `users` table (Better Auth requires it).
   - Add `sessions` table: id (text, primary key -- Better Auth generates string session tokens), userId (integer, references users.id with onDelete cascade), token (text, unique -- the session token), expiresAt (timestamp with timezone), ipAddress (text, nullable), userAgent (text, nullable), createdAt/updatedAt timestamps.
   - Add `accounts` table: id (text, primary key), userId (integer, references users.id with onDelete cascade), accountId (text), providerId (text), accessToken (text, nullable), refreshToken (text, nullable), accessTokenExpiresAt (timestamp, nullable), refreshTokenExpiresAt (timestamp, nullable), scope (text, nullable), idToken (text, nullable), password (text, nullable), createdAt/updatedAt timestamps.
   - Add `verifications` table: id (text, primary key), identifier (text), value (text), expiresAt (timestamp with timezone), createdAt (timestamp, nullable), updatedAt (timestamp, nullable).
   - Use plural table names (sessions, accounts, verifications) to match `usePlural: true` adapter config.
   - Add appropriate indexes: sessions(userId), sessions(token unique), accounts(userId).

4. Keep all existing tables (monitors, heartbeats, etc.) untouched. Only add to the users table and create new auth tables.

Important: Better Auth's session table uses `text` for the `id` and `token` columns because it generates its own string identifiers. The `userId` foreign key must be `integer` to match the existing users table. Do NOT change the users table ID type.

Run `npm run lint` after changes to ensure Biome formatting is correct.
  </action>
