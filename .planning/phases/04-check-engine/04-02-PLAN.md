  <name>Task 1: Implement check engine tick loop and heartbeat storage with tests</name>
  <files>worker/src/check-engine.ts, tests/worker/check-engine.test.ts</files>
  <behavior>
    Tick loop tests (tests/worker/check-engine.test.ts):
    - tick() queries only active monitors where lastCheckedAt is null or elapsed >= checkIntervalSeconds
    - tick() does nothing when no monitors are due
    - tick() dispatches probeHttp for HTTP monitors, probeTcp for TCP monitors, probeSsl for SSL monitors
    - After successful check: inserts heartbeat with status=1, responseTimeMs, statusCode (HTTP), updates monitor lastCheckedAt, resets consecutiveFailures to 0
    - After failed check: inserts heartbeat with status=0, error message, increments consecutiveFailures by 1
    - After 3 consecutive failures: consecutiveFailures equals 3 (no incident creation in this phase)
    - All due monitors are checked in parallel via Promise.allSettled
    - Unexpected probe rejections are logged but do not crash the tick
  </behavior>
  <action>
Create `worker/src/check-engine.ts` with these exports:

1. `startCheckEngine(db: WorkerDb): NodeJS.Timeout` -- sets up a 30-second setInterval that calls tick(). Also runs an immediate first tick. Returns the interval ID for graceful shutdown.

2. Internal `tick(db)` function:
   - Query monitors where `status = 'active'` AND (`lastCheckedAt IS NULL` OR `lastCheckedAt <= NOW() - MAKE_INTERVAL(secs => checkIntervalSeconds)`)
   - If no due monitors, return early
   - Log the count: `[worker] Checking N monitor(s)`
   - Run `Promise.allSettled(dueMonitors.map(m => checkMonitor(db, m)))` -- all in parallel
   - Log any rejected results (should not happen since probes always resolve)

3. Internal `checkMonitor(db, monitor)` function:
   - Switch on monitor.type to dispatch to correct probe (probeHttp, probeTcp, probeSsl)
   - For TCP: use parseTcpTarget(monitor.url) to extract host/port
   - For SSL: use parseHostname(monitor.url), port is always 443
   - For HTTP: pass monitor.url, monitor.timeoutMs, monitor.expectedStatusCode ?? 200
   - Insert heartbeat record with all ProbeResult fields
   - Update monitor: set lastCheckedAt to now, set consecutiveFailures to 0 on success or monitor.consecutiveFailures + 1 on failure, set updatedAt
   - Call upsertHourlyRollup(db, monitor.id, result, checkedAt) after heartbeat insert

Use Drizzle ORM operators: `and`, `eq`, `or`, `isNull`, `lte`, `sql` from `drizzle-orm`.

For the SQL interval comparison, use: `sql\`NOW() - MAKE_INTERVAL(secs => ${monitors.checkIntervalSeconds})\``

Tests: Mock the DB (all Drizzle operations) and probe functions. Use `// @vitest-environment node`. Test the logic of tick() and checkMonitor() independently. Verify correct probe dispatch by monitor type. Verify heartbeat insert fields. Verify consecutiveFailures increment/reset logic.

  <files>worker/src/rollup.ts, tests/worker/rollup.test.ts</files>
  <behavior>
    Rollup tests (tests/worker/rollup.test.ts):
    - upsertHourlyRollup truncates checkedAt to the hour boundary (minutes, seconds, ms set to 0)
    - upsertHourlyRollup inserts a new row when no existing row for that monitor+hour
    - upsertHourlyRollup updates totalChecks, successfulChecks, avg/min/max response times on conflict
    - Running average formula: newAvg = (oldAvg * oldCount + newValue) / (oldCount + 1)
    - aggregateDailyRollups groups hourly data by monitor+date into daily rows
    - cleanupOldHeartbeats deletes heartbeats older than 7 days
  </behavior>
  <action>
Create `worker/src/rollup.ts` with these exports:

1. `upsertHourlyRollup(db: WorkerDb, monitorId: number, result: ProbeResult, checkedAt: Date): Promise<void>`:
   - Truncate checkedAt to hour boundary: `new Date(checkedAt); hourBucket.setMinutes(0, 0, 0)`
   - Use `db.insert(heartbeatsHourly).values({...}).onConflictDoUpdate({ target: [heartbeatsHourly.monitorId, heartbeatsHourly.hour], set: {...} })`
   - Running average in SQL: `(avgResponseTimeMs * totalChecks + newValue) / (totalChecks + 1)` -- use `sql` template for the arithmetic
   - Min/max via SQL `LEAST`/`GREATEST`
   - Increment totalChecks and successfulChecks

2. `startMaintenanceJob(db: WorkerDb): NodeJS.Timeout`:
   - `setInterval` at 4 hours (4 * 60 * 60 * 1000)
   - Calls `aggregateDailyRollups(db)` then `cleanupOldHeartbeats(db)`
   - Wrapped in try/catch, logs errors without crashing

3. Internal `aggregateDailyRollups(db)`:
   - Query heartbeatsHourly, group by monitorId + date (truncate hour to date)
   - For each group: SUM totalChecks, SUM successfulChecks, weighted AVG responseTime, MIN min, MAX max
   - Compute uptimePercentage = successfulChecks / totalChecks
   - Upsert into heartbeatsDaily (ON CONFLICT on monitorId + date)
   - Use raw SQL via `db.execute(sql\`...\`)` if Drizzle's query builder is too awkward for the GROUP BY + upsert combo

4. Internal `cleanupOldHeartbeats(db)`:
   - Delete from heartbeats where `checkedAt < NOW() - INTERVAL '7 days'`
   - Log count of deleted records
